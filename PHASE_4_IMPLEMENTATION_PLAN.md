## Overview

Phase 4 builds the final UI component to display the structured 6-section workouts generated by the AI Coach in Phase 3. This component transforms the JSON workout data into a beautiful, interactive, mobile-responsive display.

### Key Goals
1. Display all 6 workout sections clearly
2. Show exercises with descriptions and form tips
3. Display summary statistics (duration, intensity, calories, difficulty)
4. Mobile-optimized responsive design
5. Interactive features (expand/collapse, save, share)
6. Integrate seamlessly with existing AI Coach flow

## Components to Create

### 1. **WorkoutDisplay.js** (Main Container - 150 lines)
**Purpose**: Parent component that renders entire workout
**Props**:
- `workout`: Full 6-section workout object
- `user`: User data
- `onClose`: Callback to close component
- `onStart`: Optional callback to start workout timer
- `onSave`: Optional callback to save workout

**State**:
- `expandedSections`: Object tracking which sections are expanded
- `isSaving`: Boolean for save loading state

**Functions**:
- `toggleSection(sectionName)`: Toggle expand/collapse
- `handleSave()`: Save workout to user's library
- `handleStart()`: Start workout timer/tracker
- `handleShare()`: Share workout (future feature)

**Renders**:
- Header with title and summary
- 6 section cards
- Action buttons (Start, Save, Share, Close)

### 2. **SectionCard.js** (Reusable Section Card - 120 lines)
**Purpose**: Render individual workout section
**Props**:
- `section`: Section data (warm_up, strength, etc)
- `sectionName`: Name for styling/ID
- `isExpanded`: Boolean for expand state
- `onToggleExpand`: Callback to toggle

**Features**:
- Color-coded by section type
- Duration badge
- Expand/collapse animation
- Exercise list rendering
- Sets/reps display (strength only)
- Section notes

### 3. **ExerciseList.js** (Exercise Renderer - 80 lines)
**Purpose**: Render list of exercises with descriptions
**Props**:
- `exercises`: Array of exercise names/descriptions
- `setsReps`: Optional sets/reps string (strength)
- `notes`: Optional section notes

**Features**:
- Numbered or bulleted list
- Optional form tips/descriptions
- Sets/reps guidance
- Clean, scannable layout

### 4. **WorkoutSummary.js** (Summary Stats - 100 lines)
**Purpose**: Display workout statistics
**Props**:
- `summary`: Summary object with duration, intensity, calories, difficulty

**Features**:
- Duration display with icon
- Intensity level with color
- Calories burned estimate
- Difficulty rating (visual stars or numeric)
- Clean stats grid layout

## Key Features & UX Decisions

### Expand/Collapse Strategy
- **Why**: Long lists can be overwhelming
- **How**: Click to expand, collapse by default except first section
- **Benefit**: Users can focus on current section, see full plan if needed

### Color Coding
- **Why**: Visual scannability and intensity indication
- **How**: Each section type has distinct color
- **Benefit**: Users quickly understand intensity/purpose of each section

### Mobile-First Design
- **Why**: Many users exercise with phone in hand
- **How**: Full-width cards, large buttons, readable fonts
- **Benefit**: Easy to follow during workout

### Summary Statistics
- **Why**: Users want quick workout overview
- **How**: Display total time, intensity, calories, difficulty
- **Benefit**: Quick decision on whether to do workout

### Smooth Animations
- **Why**: Professional appearance
- **How**: CSS transitions on expand, button hover effects
- **Benefit**: Satisfying, modern feel
